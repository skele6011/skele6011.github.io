IoT Smart Alarm Clock

A standalone, internet-connected hardware application built from the ground up to aggregate utility metrics cleanly without external application dependencies.

Project Overview

The goal of this project was to create a reliable desktop hardware unit capable of pulling real-time environmental data over network sockets while maintaining a completely interactive, lag-free UI on a hardware bus. Instead of relying on a pre-built commercial device, I handled everything from the core software runtime down to the manual GPIO pinout configurations.

Software Architecture & API Resilience

Operating directly on an ESP32 microcontroller meant designing code with optimization in mind. To ensure stable runtime behavior, I implemented the following software solutions:

  • Failsafe API Rotation: Real-time clocks and weather systems are prone to network request blocks if rate limits are exceeded. I engineered a request-pooling abstraction that dynamically alternates between endpoints, protecting against application hangs and IP rate limits.
  • Display Rendering Framework: Designed screen elements using the Lopaka wireframe environment and realized the physical layout using the TFT_eSPI library. Frame structures were configured to execute over parallel pipelines to eliminate display tearing and flashing.

★ Upstream Open-Source Contribution

While building out the display stack, I encountered a critical low-level processor compatibility issue within the core library infrastructure. Instead of finding a temporary workaround, I isolated the issue in the repository source code, patched the compatibility mismatch, and submitted a formal pull request directly to TFT_eSPI (an open-source project with over 4.8k GitHub stars).

Hardware Configuration

The physical infrastructure relies on explicit component grouping to manage high-efficiency inputs and visual output streams without degrading internal module performance.

Component Interface Details
ESP32D Module Primary SoC processor handling computational logic and Wi-Fi stack connection.
8-Bit Bus Parallel LCD Screen Mapped manually across precise GPIO pins to sustain higher frame-rate outputs.
Active Buzzer Integrated for immediate physical sound processing, with hardware updates underway for dynamic audio.
